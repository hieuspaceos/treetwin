# Landing Page Design System (v2.6.0)

## Per-Page Design Customization

Each landing page supports custom design via CSS variables (`--lp-*` tokens).

### Design Customization Fields

- **Preset:** Select from 6 presets (clean-light, modern-dark, gradient-bold, startup-fresh, corporate-trust, warm-sunset)
- **Colors:** Primary, secondary, accent (hex or RGB)
- **Fonts:** Headings font, body font, border-radius
- **Google Fonts:** Auto-load from Google Fonts API (no external CDN)

### Implementation

- Design config stored in landing YAML: `metadata.design`
- Section components use `--lp-*` CSS vars instead of hardcoded colors
- Presets pre-define color schemes + font pairs
- Live preview updates without save

**Admin UI:** `/admin/landing/[slug]/design` panel with preset picker + custom editor

## CSS Variables

All landing section components use `--lp-*` variables:

```css
--lp-primary: #3b82f6;
--lp-secondary: #ec4899;
--lp-accent: #f59e0b;
--lp-font-heading: 'Poppins', sans-serif;
--lp-font-body: 'Inter', sans-serif;
--lp-radius: 0.5rem;
--lp-spacing-base: 1rem;
```

## Layout System v2 (Full-Width Sections)

**Released:** 2026-03-29

All landing page sections now render full-width with edge-to-edge background support and scoped CSS styling.

### Structure

```
.landing-section-wrapper (100vw, full-width, data-section attribute)
  ├─ background color/image (covers full width)
  └─ .landing-section (max-width: 72rem, auto margins)
      └─ section content (padding controls spacing)
```

### Key Changes

| Aspect | Before | After |
|--------|--------|-------|
| **Section width** | Constrained to container max-width | Full-width (100vw) |
| **Background** | Limited by container | Full viewport width |
| **Padding** | Wrapper padding caused gaps | Inner section padding only |
| **Styling** | Hardcoded colors in components | Design variables + scoped CSS |
| **Data attribute** | None | `data-section={sectionId}` for scoped CSS matching |

### Scoped CSS

Each section can include a `scopedCss` array (auto-generated by AI clone post-processors):

```json
{
  "type": "features",
  "scopedCss": [
    { "selector": ".feature-card", "css": "background: rgba(var(--lp-accent-rgb), 0.1);" },
    { "selector": ".feature-icon", "css": "color: var(--lp-accent);" }
  ]
}
```

Rendered as:
```html
<style scoped>
  [data-section="section-features"] .feature-card { ... }
  [data-section="section-features"] .feature-icon { ... }
</style>
```

### Design Variables (No Hardcoding)

Component CSS uses only:
- Layout properties (flex, grid, width, height, padding, margin)
- Animation/transition declarations
- Semantic utilities (card mode, hover states)

**Colors flow from:** `--lp-primary`, `--lp-secondary`, `--lp-accent`, `--lp-text`, `--lp-text-muted`

**Example (bad):**
```css
/* Don't do this */
.button { background: #3b82f6; }
```

**Example (good):**
```css
/* Do this */
.button { background: var(--lp-primary); }
```

### Responsive Breakpoints

CSS Grid/Flex layouts adjust per breakpoint:

| Device | Max-width | Columns |
|--------|-----------|---------|
| Mobile | 375px | 1-2 |
| Tablet | 768px | 2-3 |
| Desktop | 1024px+ | 3-5 |

### Migration (Old → New)

If landing page was created before v3.1.0, add to each section:

```yaml
style: {}  # For future design overrides
scopedCss: []  # For auto-generated CSS
```

No rendering changes needed — backward compatible.

## AI Landing Page Cloner (v2.7.0 & v2.8.0 Post-Processing)

**Feature:** Paste URL → AI analyzes HTML → extracts sections + design → auto-fixes cloned content → generates landing config

### Post-Processing Auto-Fixes (v2.8.0)

After Gemini clone completes, apply 11 intelligent post-processors:

| Fix | Issue | Solution |
|-----|-------|----------|
| Hero Background | CSS `background-image` pollutes data | Extract JPG URL to `images.bg`, clear style |
| Subheadline Clean | Raw form syntax, HTML fragments | Strip non-semantic text |
| Font Mapping | Non-Google fonts (Arial, Verdana) | Map to Google equivalents (Roboto, Lato) |
| TopBar Icon | Font Awesome classes (fa-phone) | Convert to emoji (📞) |
| SocialLinks Icon | Icon names (twitter, linkedin) | Convert to emoji (𝕏, in) |
| Scoped CSS | No visual fidelity after clone | Inject section-level CSS (Dancing Script for cursive, accent buttons) |
| Nav Logo | Logo buried in HTML | Auto-find image URL, populate `nav.logo.image` |
| Testimonial Card | Dark bg → poor text readability | Switch card mode to light bg |
| Design Colors | Incorrect/missing primary/accent | Extract from nav/CTAs, fix textMuted contrast |
| High Contrast | Inverted contrast (dark bg + white text) | Auto-correct or remove problematic combinations |
| Broken Colors | Hex "#" without digits | Clean/validate or fallback to preset |

All fixes logged to section metadata for audit trail.

### Endpoint

```
POST /api/admin/landing/clone
Content-Type: application/json

{
  "url": "https://example.com"
}

Response:
{
  "landing": {
    "slug": "cloned-landing",
    "title": "...",
    "sections": [...],
    "metadata": { "design": {...} }
  },
  "design": {
    "preset": "modern-dark",
    "colors": {...},
    "fonts": {...}
  }
}
```

### Model

**Gemini 2.5 Flash** — Fast HTML analysis + section extraction

### Flow

1. User pastes URL in clone modal
2. Gemini analyzes HTML structure:
   - Extracts headings, sections, layout hierarchy
   - Identifies colors, fonts, visual patterns
   - Maps elements to landing section types
3. Returns:
   - YAML config with section data
   - Design preset matches + custom color values
4. User can edit before saving
5. Saved as draft for manual review

**API route:** `src/pages/api/admin/landing/clone.ts`
**Model:** `gemini-2.5-flash`
